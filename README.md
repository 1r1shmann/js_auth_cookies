# JS Auth Cookies

Небольшой учебный проект с простой авторизацией на чистом JavaScript через cookies.

Без базы данных, без PHP, без JWT, без фреймворков. Просто демонстрация того, как можно организовать примитивную систему входа в аккаунт средствами браузера.

## Для чего это было сделано

В первую очередь ради практики и экспериментов с клиентской авторизацией.

Проект показывает:

- работу с cookies;
- хранение данных пользователя в браузере;
- проверку авторизации при загрузке страниц;
- ограничение доступа к защищённым разделам;
- базовую логику входа и выхода из аккаунта.

## Возможности

- регистрация пользователя;
- авторизация через логин и пароль;
- хранение данных в cookies;
- автоматическая проверка статуса авторизации;
- выход из аккаунта;
- защита страниц от неавторизованного доступа.

## Важно

Это демонстрационный проект и не предназначен для использования в реальных системах авторизации.

Хранение чувствительных данных в cookies на стороне клиента не обеспечивает необходимый уровень безопасности для боевых проектов. Cookies часто используются для хранения данных сессии, но полноценная авторизация обычно требует серверной проверки и дополнительных механизмов защиты. :contentReference[oaicite:0]{index=0}

## Зачем использовать тогда?

Чтобы разобраться:

- как работают cookies;
- как браузер хранит данные пользователя;
- как устроена базовая логика авторизации;
- почему в реальных проектах всё обычно делают через сервер и сессии.
